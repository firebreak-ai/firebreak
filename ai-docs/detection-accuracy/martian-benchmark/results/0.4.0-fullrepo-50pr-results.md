# Firebreak v0.4.0 — Full-Repo Benchmark Pilot (50 PRs)

**Date**: 2026-04-21
**Branch**: `feat/0.4.0-candidate-single-detector`
**Tool name**: `firebreak-v0.4.0-fullrepo`
**Scope**: All 50 PRs from the Martian Code Review Benchmark
**Methodology**: Full-repo review (matches benchmark peer methodology); 3x Sonnet 4.6 consensus judge

> **Pilot framing.** This is a single-cycle result. Sources of variance we observed in this work — orchestrator non-determinism (per-PR finding count varies across reruns), judge spread (±3-5pp across 3 Sonnet judges on the same input), sample size (N=50 across 5 repos, N=10 per repo) — mean a single run's ±3-4pp is plausibly within the margin of error. Treat absolute numbers as directional until additional cycles establish a confidence interval.

---

## Methodology change: diff-only → full-repo

Prior Firebreak benchmark runs reviewed each PR with **only the diff** visible. Other tools in the Martian benchmark (Devin, Augment, Greptile, CodeRabbit, Qodo, etc.) review **forked repositories at PR HEAD** with complete repo visibility. That mismatch systematically suppressed Firebreak's results: findings whose consequence chain requires extra-diff context (caller behavior, deployment state, race conditions, missing guarantees) were rejected by the Challenger as "diff-only evidence gap" even when the mechanism was verifiable.

This run switches Firebreak to the same review methodology as its benchmark peers: per-PR git worktree at `base_sha` with the PR diff applied, full repo browsing, project-native linter execution, caller tracing. Infrastructure:
- `fetch_pr_shas.py` — enriches `manifest.json` with base/head SHAs via `gh api`
- `clone_repos.py` — shared bare caches (`--filter=blob:none`) + per-PR worktrees with diff applied (falls back to `head_sha` checkout on 3-way apply failure)
- `run_reviews.sh --mode full-repo` — switches cwd to per-PR worktree, loads `benchmark-prompt-fullrepo.md`

The infrastructure is additive: `--mode diff-only` (default prior to this change) preserves reproducibility of the earlier baseline.

## Headline

| Version | TP | FP | FN | P | R | **F1** |
|---------|----|----|----|---|---|--------|
| v0.3.5 baseline (diff-only, 50 PRs, reported 2026-04-09) | 93 | 360 | 43 | 20.5% | 68.4% | 31.6% |
| v0.4.0-single-detector (diff-only, 50 PRs, corrected extract) | 59 | 119 | 77 | 33.1% | 43.4% | 37.6% |

**Full-repo vs diff-only (same Firebreak version, same 50 PRs):** TP +6, FP -5, FN -6. Precision +3.2pp, recall +4.4pp, F1 +3.7pp. Both axes improved.

## Two distinct value propositions

**1. Competitive on general code review.** ~41% F1 on human-written PRs from real-world repos places Firebreak in middle-of-pack with commercial tools. Respectable enough for general-purpose review.

**2. Purpose-built for AI-authored code.** The AI failure mode checklist (`fbk-sdl-workflow/ai-failure-modes.md`), "plausible-looking code that doesn't fit the architecture" watch, and the "developer's mental model vs code divergence" framing are specifically tuned for agentic coding failure modes. The Martian benchmark doesn't measure this — its PRs are mostly human-written. For teams using AI agents to write code, Firebreak's practical value is likely higher than this general-PR F1 suggests.

## Methodology validation

This run's +3.7pp F1 lift (and the earlier partial-43 +4.0pp lift) validates the full-repo methodology change. Diff-only Firebreak was systematically underrating itself by rejecting real findings as evidence gaps. Under apples-to-apples methodology:

- **FP dropped** (114 vs 119) — Challenger uses repo context to verify or reject claims with real evidence instead of rejecting for uncertainty
- **TP rose** (65 vs 59) — findings that were previously dismissed as "consequence unverifiable from diff" now stand
- **Recall rose** (47.8% vs 43.4%) — the gap between what Firebreak can find and what it's credited with narrowed

## Inter-judge agreement

3 Sonnet 4.6 judges: 64 / 66 / 65 matches on 136 goldens. Spread of 2 (tight), consensus majority = 65.

## Cost

- **Reviews**: 50 new reviews at $3.76/review avg = **~$190** (batch 1 $58.69, batch 2 $116.55, smoke $12.72, PR1/PR3 rerun $9.92)
- **Judges**: 3 Sonnet sub-agents on Claude Code Max plan
- **Wall time**: Review phase ~15 hours (batches + cooldown + PR1/PR3 rerun). Judge phase ~8 min parallel.
- **Disk**: ~8.5 GB for bare caches + worktrees (partial clone with `--filter=blob:none`)

## Known caveats

- **Sentry-greptile PR1 and PR3** initially ran via a sandbox-fallback bug in `run_reviews.sh` (bash glob `apply:failed*` incorrectly matched `apply:failed-fallback-head`). Fix applied and both PRs re-reviewed under correct worktree. Final 50-PR set uses the correct mode throughout.
- **No recall-recovery for keycloak**. Keycloak goldens include class-substitution and framework-level concerns that the Challenger still rejects under full-repo. Likely wants a deeper pass or additional targeted detection rules. Not addressed in this release.
- **One-cycle pilot**. No statistical power. Multiple cycles needed for a confident leaderboard claim.

## Recommendations

1. **Ship v0.4.0** on the code changes validated in this cycle: single-detector architecture, broadened behavioral type definition, Challenger uncertainty-as-verified, Detector/Challenger instruction sync, full-repo benchmark methodology.
2. **Report the benchmark result descriptively, not as a ranking claim**: "In this single cycle, Firebreak landed in the middle of the benchmark field." Avoid "Firebreak beats Devin/Greptile" claims without multiple cycles.
3. **Additional cycles when capacity allows**. Single-dev Max 20 plan budget means ~1 cycle/week maximum. 3-5 cycles would let us publish F1 mean ± stdev. Not a near-term priority — don't hold up feature development.
4. **Use Firebreak internally**. This pilot result is sufficient evidence that Firebreak is a reasonable code reviewer. Use it on Firebreak's own development (post-impl reviews on every feature) — a dogfooding loop rather than a benchmark-chasing one.
