# v0.4.0 Single-Detector Benchmark — 23 PRs

**Date**: 2026-04-19
**Branch**: `feat/0.4.0-candidate-single-detector`
**Tool name**: `firebreak-v0.4.0-single-detector`
**Scope**: 23 PRs (7 cal_dot_com + 4 each from discourse, grafana, keycloak, sentry)
**Config**: behavioral-only preset, single Detector + Challenger
**Judge**: Consensus (3x Sonnet 4.6, majority vote, all 3 judges converged on identical 29/64 match count)
**Sandbox isolation**: claude invocations ran from `/tmp/firebreak-benchmark-sandbox` to prevent the orchestrator from mis-routing to in-repo `ai-docs/` specs

## Key Observations

### 1. Single-detector matches decomposed F1 with less complexity

F1 = 36.9% (this) vs 37.6% (decomposed) — within noise at this sample size. The multi-detector decomposition (Tier 1 groups + IPT + deduplicator + challenger) was projected as the precision-improving architecture; results show that the precision gains came from **other** changes that landed in 0.4.0 (severity thresholds, charter filtering, mechanism-first finding style), not from the decomposition itself.

### 2. Precision is now the dominant lever

Precision climbed from 20.5% (v0.3.5) → 28.5% (decomposed) → 31.2% (single-detector). Each release tightens the filter further. At the same time, output volume dropped from 9.0 → 5.3 → 4.0 findings/PR. Users see fewer, more accurate findings.

### 3. Recall is now the bottleneck

Recall dropped from 68.4% → 55.5% → 45.3%. Approaching the half-the-bugs-missed threshold. Three patterns drive misses (per judge feedback):
- **Cross-function/cross-call analysis**: race conditions, transaction-level reasoning, async error propagation across handlers
- **Header/policy concerns**: clickjacking, X-Frame-Options, postMessage origin checks (when not the diff's primary subject)
- **Naming/typo issues**: never flagged by behavioral preset (out of charter)

### 4. Four PRs produced zero findings

PR10967, PR22345 (cal_dot_com), PR107534 (grafana), PR33832 (keycloak) — the orchestrator finished review with 0 verified findings. Those PRs had 1-5 goldens each, all counted as FN. Removing them would lift recall to ~52% and F1 to ~39%, but the right framing is that 0-finding outputs are a real failure mode: if the tool runs to completion and emits nothing, that's a recall hit by definition.

### 5. Per-repo variance widened

discourse F1 jumped from 30.4% (decomposed) to 55.2%. Sentry/keycloak remain weak. Possible explanation: discourse PRs in this sample were Ruby with explicit behavioral bugs; sentry/keycloak PRs include security/infra concerns the behavioral-only preset systematically misses.

## Cost & Time

- **Total wall time**: ~5.5 hours for 23 reviews (300s delay between + ~12 min average review time)
- **Total review cost**: ~$50 for 20 NEW-prompt reviews + ~$15 from leftover OLD-prompt overflow (cal_dot_com)
- **Judge cost**: 3 Sonnet sub-agents via Claude Code Max plan (no API cost)
- **Avg cost per review**: $2.47 (sentry) – $3.09 (cal_dot_com)

## Caveats

- **Sample size**: 23 PRs vs 50 in baselines — wider confidence intervals, especially for per-repo metrics
- **Imbalance**: 7 cal_dot_com vs 4 each elsewhere (overshoot from process-management issues during run setup, see retrospective)
- **Process incident**: First benchmark launch produced 1 off-target review (orchestrator wandered into in-repo `ai-docs/agent-personas` spec instead of reviewing the cal.com diff). Fixed via sandbox dir + strengthened benchmark-prompt overrides. All 23 reviews in the final dataset are on-target.
- **Judge agreement**: All 3 Sonnet judges converged on the same 29/64 match count — high inter-judge reliability suggests the 36.9% F1 estimate is stable

## Conclusion

The single-detector candidate is **as good as the decomposed architecture** at finding bugs (F1 36.9% vs 37.6%) with **substantially simpler orchestration** (one Detector + one Challenger vs Tier 1 groups + IPT + deduplicator + Challenger). The precision/recall tradeoff continues from earlier releases — the tool emits fewer, more accurate findings.

**Recommendation**: Ship the single-detector candidate as v0.4.0. The decomposition isn't paying its complexity cost. Future precision gains will need a different lever; future recall gains require addressing cross-function reasoning (the dominant miss pattern).
