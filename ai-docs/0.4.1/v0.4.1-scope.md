# v0.4.1 Scope

Planned features for v0.4.1. Each item depends on v0.4.0 (detector-decomposition) being complete and evaluated.

## tiered-detection

Tier 2 cross-module detection using AST skeletons or behavioral summaries. Extends the decomposed Tier 1 architecture with cross-file analysis beyond the Intent Path Tracer's scope.

- Cross-module enhancement for unbounded data structure growth (trace resource lifecycle across modules — writes without corresponding cleanup jobs)
- Spec not yet started; depends on v0.4.0 orchestrator infrastructure

## output-verbosity

Findings-only production mode to reduce output ~60-70% while preserving full diagnostic output for benchmark/evaluation runs.

- Intent register and retrospective are diagnostic scaffolding that inflates both output and input tokens (context accumulates across detection rounds)
- Requires v0.4.0 retrospective data (AC-15 metrics over multiple runs) to determine which diagnostic fields are load-bearing for ongoing evaluation vs one-time validation
- Design verbosity modes after establishing which retrospective fields are safe to strip
- Benchmark data: competing PR-level tools produce only inline comments; full Firebreak output includes intent register, finding writeups, findings summary, and retrospective
