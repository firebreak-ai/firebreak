# 0.3.3 Self-Improvement Cycle

Combined self-improvement proposals from Phase 6 remediation completion and post-remediation full-project code review. 54 proposals targeting 11 Firebreak assets, zero duplicates after deduplication analysis.

**Sources:**
- Phase 6 remediation retrospective (11 proposals from `/fbk-improve`)
- Post-remediation full-codebase code review (43 proposals from 8 improvement analysts)

**Key theme:** The AI failure mode taxonomy effectively catches how AI code specifically fails, but creates a blind spot for standard engineering issues that any competent reviewer would flag. Proposals 21-31 expand the detection scope to cover general engineering concerns alongside AI-specific failure modes.

## Coordinated Changes

Several proposals target the same underlying concern at different pipeline layers. These should be applied together:

**Dead infrastructure** (proposals 25, 32, 37, 47): Detection instruction in code-review-guide.md, structural detection target in quality-detection.md, checklist item in ai-failure-modes.md, reference instruction in existing-code-review.md.

**String-based error classification** (proposals 35, 42, 46): Detection target in quality-detection.md, checklist item in ai-failure-modes.md, detection instruction in existing-code-review.md.

**Non-enforcing tests** (proposals 24, 38, 49-53): Expanded test-integrity definition in code-review-guide.md, expanded checklist item in ai-failure-modes.md, and 6 new test reviewer criteria/checks.

**Severity tiering** (proposals 10, 12, 48): Severity signal in detector output, severity classification in challenger output, severity ordering in review presentation.

## Impact Assessment

**Highest impact — expanding detection scope (proposals 32-35, 36-42, 43-48):** Close the gap between AI-specific failure mode detection and general engineering review. 55% of post-remediation findings were standard engineering issues outside the current taxonomy.

**High impact — severity and structure (proposals 10-13, 18, 20, 30, 48):** Severity tiering, cross-cutting pattern grouping, adjacent observations, and comparative analysis transform review output from a flat findings list into actionable intelligence.

**High impact — test reviewer expansion (proposals 49-54):** Address the pipeline's blind spot for test-integrity issues in its own output. 4 of 6 remediation-caused findings were test-integrity issues the per-phase test reviewer didn't catch.

**Moderate impact — operational improvements (proposals 1-8, 16-17, 19, 21-22, 26-29):** Process fixes (hook retry cap, foreground execution, stuck-agent recovery, parallel spawning), guide clarifications (nit exclusion, origin guidance), and implementation quality (per-site completion conditions, E2E exceptions). Individually small; collectively they reduce friction and close minor gaps.
