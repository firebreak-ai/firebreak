For context asset authoring principles, read `home/dot-claude/docs/context-assets.md`.

Apply the Necessity Test to every instruction: **"If removed, is the agent more likely to make a mistake?"**

Check every context asset for:
- Preambles or introductions — start with the first instruction
- Passive or third-person framing — use direct-address imperatives
- Negative framing — state what to do, pair prohibitions with the positive alternative
- Compound instructions — each instruction covers one verifiable constraint

Review heuristic: **"Is this sentence written for the agent, or for a human browsing the repo?"**
